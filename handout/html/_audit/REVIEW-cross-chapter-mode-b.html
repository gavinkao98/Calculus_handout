一、方法與範圍

對 handout/fragments/ch01–ch04/sec-*.html 全 20 節做跨章一致性走查，鎖定使用者指定的兩類發現：

  • 引用錯誤——每個「Theorem N.M / §X.Y / Example X.Y / Figure X.Y / Chapter N」式引用是否對應到存在且型別正確的物件（CONTENT_SPEC §6/§15 手動編號一致性）：dangling、wrong-type、wrong-number（改版漂移）、content-mismatch、未兌現的 forward-ref。
  • 矛盾——跨章記號衝突、核心定義（函數／一對一／反函數／極限／連續／可微／指對數／級數收斂）不一致、術語漂移、跨章主張或承諾互相牴觸。

流程：每節抽取「編號物件＋跨章引用＋定義/記號慣例」→ 合併成全域索引 → 逐章引用解析器＋4 個主題式矛盾獵手產候選 → 每個候選由敵對驗證者重讀原檔複核（預設 refute，避免 over-report）→ 完整性批判補漏再驗證。共 37 個 subagent。
