二、同批修 B.1–B.5：三項真缺陷

FIX-1已修Prop B.5（鴿籠）補量詞　sec-b-5.html

For every positive integer \(n\), if \(n + 1\) objects are distributed among \(n\) boxes, then at least one box contains two or more objects.

全附錄唯一有自由變數卻無量詞的命題，而 Prop B.4 已用同一句式、Strategy B.1 第 1 步正是「Find the quantifiers」。\(n \ge 1\) 數學上也真的需要（\(n=0\) 使證明的 \(1+\cdots+1=n\) 無意義）。

FIX-2已修（最小版）Example B.3 的 \(N\) 限正（3 處）　sec-b-3.html：題幹／Solution bullet／組裝後的否定式

序列自 \(a_1\) 起，\(N \le 0\) 會指到不存在的項。但這是文獻通例（Rudin 即寫 “there is an integer \(N\)…”），屬「比通例再嚴一格」，非改錯。意見書的 \(N\in\mathbb N\) 版本已拒——集合記號不是 SPEC §3 的 Stewart 語域，且意見書自己也寫「不要把附錄改成離散數學章節」。此拒絕後來成了本輪的一致性標尺（見第五節 \(\mid\)）。

FIX-3已修（三輪後回到最初建議）§B.4「\(3\varepsilon\)」正名　sec-b-4.html Strategy B.4 bullet 2

The “\(3\varepsilon\)” you see in print is this move with \(k = 3\).
The “\(\varepsilon/3\) trick” you see in print is this move with \(k = 3\).

不是數學錯——「3ε argument」是真實印刷慣例；是把姊妹招式的名字掛到本招式上（本 bullet 描述的動作是「除以 \(k\)」＝ε/3 trick，而寫「\(<3\varepsilon\)」的作者恰恰沒有除）。本輪最大的自我修正見第六節。

結論——路走完了：解釋它 → 引入未建立的邏輯步；只點名 → 製造答不了的疑問。這個提及在這個閱讀水平上救不回來。第三方意見最初的建議（單純改名為 ε/3 trick）從頭就是對的；我把一個一字修正搞成三輪。已全砍，並在 sec-b-4.html 的註解留下「DO NOT re-add」的約束＋兩次失敗的理由，免得未來有人「好心」再加回去。
閘之間的分歧記此：散文閘主張留著、只解 garden-path；三份 sim 主張刪。兩者審的不是同一件事——散文閘審那句好不好讀，sim 審讀者能不能接受那個斷言。PIPELINE.md §易懂性單一錨明訂「blocking 主證據＝learner-sim 盲測，S·A·V 維度 A 降為上游預篩」，故以 sim 為錨。Codex R2 亦判精簡版「no false conclusion」——不衝突：它審的是「是否為假」，sim 審的是「是否讀得下去」。
