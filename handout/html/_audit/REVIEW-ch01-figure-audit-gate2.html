① Blocking — 已修＋回歸

Fig 1.20 squeeze-x2sin · D6 · Blocking

Codex 指控：PNG p34 在原點畫出實心點。FIGS 於 chapter1-print-standalone.html:2524 將曲線分成 \([-1.1,-0.01]\)、\([0.01,1.1]\) 後又加入 { type: "dot", x: 0, y: 0 }（實心）。fragment sec-1-5.html:252–255 說明 bound valid for all \(x \neq 0\)，caption 稱此為 \(y = x^{2}\sin(1/x)\) 的 graph。
Codex 論點：函數 \(x^{2}\sin(1/x)\) 在 \(x = 0\) 未定義。實心點按本講義慣例（如 Fig 1.11/1.12/1.21）表示「函數值存在且等於此值」，與數學對象矛盾。
Codex 建議：移除實心點；若要標示極限值，改用空心點。
交叉比對與分析：
  • 慣例一致性：本講義確立的慣例——Fig 1.11 limit-same-near-a 中空心點 = 極限值但 \(f(a)\) 未定義或不等、實心點 = \(f(a)\) 存在；Fig 1.21 precise-limit 同。若套用同一慣例，Fig 1.20 原點的實心點暗示 \(f(0) = 0\) 已定義。
  • 數學面：原式 \(x^{2}\sin(1/x)\) 在 \(x=0\) 確實無定義。但許多教科書在此例會自然地令 \(f(0)=0\)（夾擠定理的結論正好保證極限為 0，延拓後連續）。這是一個教學意圖的選擇而非純然的數學錯誤。
  • prose 語境：fragment 的 caption 寫「The graph of \(y = x^{2}\sin(1/x)\)」，未定義 \(f(0)\)；prose 說 bound 對「all \(x \neq 0\)」成立。以 prose 為準，\(x=0\) 不在定義域，實心點確實不一致。
結論：finding 有理。但修法不只有「刪點」一種——也可以在 prose 中明確定義 \(f(0) = 0\) 使之連續（許多教材的做法），此時實心點即正確。兩種修法都合理，須使用者裁決教學意圖。
使用者裁決：方案 A — 刪除實心點。
已從 FIGS["squeeze-x2sin"] 移除 { type: "dot", x: 0, y: 0 }（chapter1-print-standalone.html:2524）。曲線仍分兩段 \([-1.1,-0.01]\)、\([0.01,1.1]\)，從兩側趨向原點但不標記函數值，與 prose「for all \(x \neq 0\)」一致。
回歸審核：
  • D6：原點不再有任何 dot 元素，不暗示 \(f(0)\) 已定義。clean
  • D5：圖與 caption「graph of \(y = x^{2}\sin(1/x)\)」一致（未定義點無標記）。clean
