① Confirmed blocking — 已修＋回歸

Fig 1.18 floor-function · D4 · NEW

結論：兩軸整數刻度 1,2,3,4 的 tick entry 缺 tex；buildPlot 的 piTex fallback 只認 \(\pi\) 倍數與 0、不認純整數，因此只畫刻度線、不出數字。讀者讀不出階高與跳點所在整數。
為何 blocking：Example 1.33 是讀值題（\(x\to3^-\) 為 2、\(x\to3^+\) 為 3）。承載教學值的軸缺刻度文字 → 讀者無法讀值，正是 rubric 明列的 D4 Blocking 案，且與舊稿 B4/B5（restricted-sine／read-limit-graph）同一缺陷類型。
證據：chapter1-print-standalone.html:2481-2482 — xticks/yticks 的 1..4 無 tex；render 後 PNG 僅 \(-1\) 有數字。
對抗式複核：UPHELD — Blocking　code＋pixel 雙重核對：tick 確無 tex、piTex 對純整數回傳 undefined、render 僅 \(-1\) 有字；非 rubric 保護的 non-finding，與逐字命中的 D4 Blocking 案吻合，維持 blocking。

修法（已套用）

於 FIGS 的 floor-function 定義為 1,2,3,4 補 tex（比照既有的 -1 entry、亦即舊稿 B4/B5 的同款補法）：
xticks: [{x:-1,tex:"-1"},{x:1,tex:"1"},{x:2,tex:"2"},{x:3,tex:"3"},{x:4,tex:"4"}]
yticks: [{y:-1,tex:"-1"},{y:1,tex:"1"},{y:2,tex:"2"},{y:3,tex:"3"},{y:4,tex:"4"}]
FIGS 位於 standalone 的 script 區、在 build.py 的 BEGIN/END-CONTENT-FRAGMENTS 替換區之外，故此修法不會被重 build 覆蓋。
回歸審核（獨立 agent）：D4 clean　兩軸整數刻度皆顯示可讀數字、Example 1.33 讀值可從圖直接取得；D2/D3/D5/D6/D7/D8 仍乾淨；本圖視覺 blocking 已歸零。回歸另提一條 D1 advisory（見下），非 blocking。

② Confirmed advisory — 已套用＋重 render 驗證

Fig 1.4 sine-not-1to1 · D4

內容：y 軸 \(\pm1\) 的 tick 無 tex（同 piTex 不認整數），故只有刻度線、無 \(1\)/\(-1\) 數字。姊妹圖 restricted-sine 已明標 tex:"-1"/"1"，故屬不一致。
為何只是 advisory：本圖教學點是「水平線 \(y=c\) 與 \(\sin x\) 多次相交」，由 \(y=0.5\) 虛線＋5 個交點＋\(x\) 軸 \(\pm\pi,\pm2\pi\) 承載；\(y\) 軸振幅 \(\pm1\) 非承載值。圖讀起來無誤。
對抗式複核：UPHELD — Advisory　確為真（與姊妹圖不一致屬實），但非承載軸 → 依 rubric 為 advisory，非 blocking、非 non-finding。建議修法（與姊妹圖一致化）：yticks:[{y:-1,tex:"-1"},{y:1,tex:"1"}]。
已套用：\(y\) 軸 \(\pm1\) 補 tex，重 render 確認 \(-1,1\) 已顯示（label dump 含 "−1","1"）。

Fig 1.18 floor-function · D1 · （回歸時新提）

內容：y 軸的 \(-1\) 刻度標籤與 \(y=-1\) 那段階梯線（\(x\in[-1,0]\)）及其右端空心點視覺相鄰／輕度重疊。
定性：此 \(-1\) ytick 在本次修法前即已存在（修前 -1 是唯一帶 tex 者；本次只新增正向 1–4），故此 crowding 為既有現象、非本次引入。首輪 audit 判 D1 clean；回歸 agent 判低優先 advisory。數字仍可辨、且 \(-1\) 非 Example 1.33 讀值所需（讀值在第一象限）。
已套用（採 b）：省略 \(y\) 軸 \(-1\) 標籤（保留 \(x\) 軸 \(-1\) 錨定負向尺度，與其他圖只標承載刻度一致）。重 render 確認 crowding 消除、無新碰撞（label dump 顯示 \(y\) 軸僅 \(1,2,3,4\)）。

③ Withdrawn — 對抗式複核判為 non-finding（6 條）

Fig 1.15 one-sided-infinite · D5 · raw Blocking → false positive

原指控：右下面板曲線在漸近線右側（\(a^+\)）但 note 標 \(a^-\)，第四案 \(a^+,-\infty\) 被錯標。
對抗式複核：non-finding　駁回——指控的關鍵證據不實。chapter1-print-standalone.html:2411 實際為 \(\lim_{x\to a^{+}} f(x)=-\infty\)（正確的 \(a^+\)）。四面板與 prose／figcaption 列舉的四案一一對應：TL \(a^-,+\infty\)／TR \(a^+,+\infty\)／BL \(a^-,-\infty\)／BR \(a^+,-\infty\)。auditor 在該 render 縮放下誤讀上標、再杜撰了與檔案矛盾的行號宣稱；其「修法」反而會植入它所指控的缺陷。（已另由人工核對 source 確認一致。）

Fig 1.3 restrict-x2 · D4

原：兩軸無數字刻度。
non-finding　非承載軸 minimalism（純定性反射示意，課文不要求讀座標），rubric 明文「非承載軸缺刻度 → advisory」且原審自標 optional；加刻度純美學，降為 non-finding。

Fig 1.16 vertical-asymptote · D3

原：\(\pm20\) 尺度下 \(y=2\) 水平漸近線刻度貼近 \(x\) 軸。
non-finding　本圖承載點是 \(x=3\) 垂直漸近線（清楚可辨）；\(y=2\) 非 figcaption／Example 1.23 教學點。把非承載 HA 的壓縮提為 finding 即 over-report。

Fig 1.20 squeeze-x2sin · D3

原：\(y\)-window 不對稱（\(-0.8..1.2\)），下包絡看似被裁。
non-finding　圖未繪 plot-box 邊框，讀者看不出「被裁」，曲線只是自然收尾；對稱性屬美學偏好，未影響可讀性。

Fig 1.20 squeeze-x2sin · D1

原：曲線 label 的 \(\tfrac{1}{x}\) 略擠。
non-finding　label 在空白處、不蓋任何元素（原審亦承認），\(1/x\) 仍可讀；字級偏好屬保護 non-finding，且換記號是 copyedit、超出視覺閘範圍。

Fig 1.21 precise-limit · D5

原：中央兩點（\((a,L)\) 空心、\(f(a)\) 實心）未直接標 f(a)，初看可能不確定。
non-finding　標準 \(\varepsilon\)-\(\delta\) 慣例（空心＝極限值、實心＝實際值）＋緊鄰散文「The value \(f(a)\) itself may be different from \(L\)」已消歧義；§10 明文「direct label permitted, not required for every mark」。原審自標 Optional／not blocking — 一條「無需動作」的條目即非缺陷。
