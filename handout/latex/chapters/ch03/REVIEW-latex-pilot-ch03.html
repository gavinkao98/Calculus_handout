1. 一句話結論

同樣的內容、同樣的 150mm 版心、同樣的視覺設計語言，LaTeX 用 24 頁排完 HTML 要 27 頁的內容 （密約 11%），且所有驗收閘全綠。密度來自 TeX 的全段斷行（Knuth–Plass）＋原生斷字＋microtype， 不是把字縮小——兩側字級與版心完全相同。

3. 需要你裁決的事項

F1 既有 HTML 瑕疵，會原樣進 PDF

composed-mapping（Figure 3.5）的標籤用 Unicode 下標零 x₀（U+2080），Inter 沒有這個字符，Chrome 就單獨對那一個字符掉到 Times New Roman。已用 CDP CSS.getPlatformFontsForNode 逐節點比對，真實 HTML 頁面與匯出的 PDF 完全一致（Inter×7 + Times×1），故非匯出失真，是既有 HTML 的既存瑕疵。依 D6「HTML 留檔不修」未動。

需要你裁決：要不要為出版級品質，把圖標籤的 x₀ 改成 MathJax 標籤（需動 standalone 的 FIGS，屬 D6 已裁定「不修」的範圍，故請你裁決）？

F2 LaTeX 側新增了 HTML 沒有的東西

ch03 的 HTML proof 完全沒有 QED 記號（6 個 proof、16 個 solution 皆無），雖然 CONTRACT-html-writing.md 說 proof 應以 <span class="qed qed-proof"> 收尾。目前 LaTeX 側也沒有補 □（跟隨 HTML 現況）。

需要你裁決：要跟隨 HTML（維持無 QED），還是取「書的樣子」補上 □？

F3 字體不同族

圖內的標籤是 Inter（由 CDN 載入、已嵌進匯出的 PDF），而圖說與 env kicker 是 NCM Sans。HTML 兩者都是 Inter。依 kickoff §4.4「裝字體屬新安裝，先問；不裝則以 NCM Sans 代替並記錄」，本輪未裝 Inter。

需要你裁決：要不要裝桌面版 Inter（新安裝，需你同意），讓 LaTeX 側的 UI 面與圖內標籤同族？

F4 pair 圖的置中基準

匯出的 page box 含標籤溢出，左右不對稱；LaTeX 置中的是墨水框，HTML 置中的是 SVG 框，故圖可能有數 px 級視覺位移。ch03 只有 remainder-tangent（Figure 3.6）一張 pair。

需要你裁決：人眼閘未發現可見差異，列此備查。

F5 圖內文字閘目前是循環的（已知極限，不擋 pilot）

check_prose.py 的 panel note 檢查，oracle 是 exporter 自己申報的 figures.json。已實測此偽陰性：把 exporter 退回只 clone <svg>，它就不申報 notes，閘於是說「無 note 需檢查」而放行——正是它要抓的那個 bug。它能守的是「申報之後」的環節（manifest → convert.py → LaTeX → PDF）。另一個做法（HTML PDF vs LaTeX PDF 詞集比對）也實測被否決：note 是 “larger h”，而 “larger” 在課文散文裡也有（the larger triangle OAC…），詞集找得到就誤判成沒掉。

需要你裁決：rollout 前要補一個獨立於 exporter 的 oracle（只讀 live page 的 CDP 探針，或對 HTML 印出的 PDF 做 exact-phrase count）。目前 ch03 的 2 條 note 已用人眼＋pdftotext 逐條確認在 PDF 裡。
