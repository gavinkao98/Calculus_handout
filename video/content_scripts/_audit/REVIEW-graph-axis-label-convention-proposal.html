結論：你的觀察成立，且有確切技術根因。但要分成兩個獨立缺口看：
  • 缺口 A（刻度／級距）：XY 軸線其實一直都有畫，缺的是刻度數字——它被硬編碼關閉，而既有的「teaching ticks」補救機制在 ch01 實測一個都沒用。
  • 缺口 B（方程式標籤位置）：placement 是啟發式 ＋ 幾乎全靠手填 label_point 硬擺（字面意義的「憑感覺」），而且 sizecheck 的重疊偵測故意豁免整個圖層，所以「標籤撞標籤／撞註解」機器完全不擋，只能等 render 後 VLM 事後抓。
我的建議優先序：先做 B.1 偵測器（高把握、外科、補真洞）→ 再評估 A 的條件式 lint 次之；B.2 自動擺位暫不做（投機、複雜）。本文件詳述根因與三種修法，供你裁決。

1 · 現況診斷（附依據）

1.1　軸線一直畫，但刻度數字被硬編碼關閉

build() 永遠建 Axes 並當 block 加入場景，預設還在軸尖標 x／y 字母。所以「沒有軸」是錯覺——真正缺的是刻度與級距數字：

axes = Axes(
    x_range=[-1.45, 1.45, 0.5],   # 預設；可被 storyboard 覆寫
    y_range=[-0.28, 1.55, 0.25],
    axis_config={
        "include_ticks":   False,   # ← 沒有刻度短線
        "include_numbers": False,   # ← 沒有級距數字
    },
)

graph_focus.py:376–391（軸建構）· :393 _add_axis_labels（x／y 字母，axis_labels: false 可關）· :385–386（刻度／數字硬關）

1.2　補救機制存在，但實測沒在用

已有 opt-in 的「teaching ticks」：在 axes 區塊寫 x_ticks: [{at, label}] 就會在指定座標放幾個關鍵刻度（例如在 $x=a$ 標 $a$、在 $y$ 軸標 $L$）。docstring 明寫用意是「讓一張沒數字的圖仍保有尺度感」。

但掃過 ch01 全章 storyboard：x_ticks／y_ticks 使用次數＝0。定位幾乎全靠 label_point。所以圖真的是「無刻度、無尺度錨點」的。

graph_focus.py:82–103 _axis_ticks（機制本體）· storyboards/ch01_inverse_trig_functions.yml（grep x_ticks/y_ticks → 0 hit）

1.3　標籤位置：啟發式 ＋ 手填座標（這就是「憑感覺」）

_place_function_label 是一條 fallback 鏈，沒有任何碰撞避讓：

label_point（顯式座標） → label_x（沿曲線某 x） → 曲線尾端 → next_to(graph, side)

實務上幾乎每個標籤都靠手填 label_point: [x, y] 硬擺。真實例子——一張直角三角形圖，三條邊的標籤座標全是肉眼喬出來避開彼此的：

- { kind: line, ..., label: '$1$',        label_point: [3.22, 0.90] }
- { kind: line, ..., label: '$3$',        label_point: [1.35, 1.12] }
- { kind: line, ..., label: '$2\sqrt{2}$', label_point: [1.45, 0.16] }
- { kind: point, ..., label: '$\theta$',  label_point: [0.55, 0.62] }

graph_focus.py:142–166 _place_function_label · storyboards/ch01_inverse_trig_functions.yml:209–212

1.4　關鍵根因：sizecheck 的重疊偵測「故意」豁免整個圖層

所有 plot block（含 label.N）在組裝時被改成 layer="graph"；而重疊偵測 _overlap_issues 只檢查 content 層。等於把圖內所有東西都排除在碰撞偵測之外。

# graph_focus.py:404–405 — 把整組 plot 標成 graph 層
for b in plot_blocks:
    b.layer = "graph"

# sizecheck.py:149 — 只查 content 層
if getattr(b, "layer", "content") != "content":
    continue   # graph 層直接跳過

本意是放行軸空間裡「合理的重合」（點落在曲線上、虛線導引穿過交點、標籤緊貼曲線）。代價是它順手把「標籤撞標籤、標籤撞註解」這類真 bug 也一起放行了——這類只能等 render 後 VLM 視覺幀稽核（V1–V8「label 不可壓線／相撞」）事後抓，再回頭手補 label_point。

註：_overflow_issues（出框檢查）對每個 block 都跑，所以標籤掉出畫面仍會被擋；只有「畫面內互相重疊」這一類是盲區。

graph_focus.py:404–405 · sizecheck.py:133–180 _overlap_issues · :63–107 _overflow_issues · DESIGN.md:364（VLM gate「graph label 不可壓線」）

2 · 缺口 A 提案 — 刻度／級距「條件式規範」

主張：不要一刀切「永遠標數字」。乾淨無數字軸是刻意的 house style（3b1b 風格），對「形狀類」圖是對的；標滿數字反而是雜訊。缺陷只出在「讀值類」圖。

圖的類型判準刻度規則例
qualitative（形狀）旁白只談形狀／趨勢／對稱／單調，不指名具體座標維持無刻度（現狀）反函數對 $y=x$ 對稱、漸近線行為
quantitative（讀值）旁白指名某座標或要讀值（「在 $x=a$」「值為 $L$」「$\arccos 1 = 0$」）必須在那些座標放 teaching-tick$\sin\theta=\tfrac13$ 求 $\tan\theta$、極限讀值

落地：

  • (A-1) 成文：把上表寫進 DESIGN.md §graph（現在只是 docstring，未成文）。
  • (A-2) lint：storyboard 旁白若出現座標引用（x=…／值為…／命名點），而對應 plot 的 axes 區塊沒有對應 x_ticks／y_ticks → warn。把「該標沒標」從靠記性變成機器提醒。

取捨／風險：A-2 的「旁白指名座標」靠正則偵測，會有偽陽／偽陰；定位成 warn（不 block render）較安全。判準本身偏內容判斷、非純工程，建議先 A-1 成文、A-2 視情況再上。

3 · 缺口 B 提案 — 標籤位置「用偵測器補洞」

B.1　收窄 graph 層豁免，補上標籤重疊偵測　建議優先做

改什麼放寬 §1.4 那條全域豁免：曲線／點／帶狀仍豁免（合理重合），但 label.N 區塊要互相查重疊、也要對 annotation.N 查重疊。

怎麼做給標籤 block 一個可辨識的子層（如 layer="graph_label"），_overlap_issues 納入 graph_label＋content 一起兩兩比 AABB，沿用既有的 OVERLAP_FRAC=0.20 門檻與 containment 例外。

為何bestrender 前、確定性、免費地擋下整類 bug，正好補掉現在只能靠 VLM 事後抓的洞。改動集中在 sizecheck.py 的層判斷 ＋ graph_focus.py 標籤 block 的 layer 賦值，外科手術級。

驗收先寫一個能重現「兩標籤重疊」的 storyboard fixture（紅燈），改完讓它亮 warn（綠燈），且既有 ch01 不誤報。

動到：sizecheck.py:133–180 · graph_focus.py:256–262 / 308–314 / 331–337（label block 的 layer）

B.2　標籤自動擺位（occupancy 取重疊最小）　暫不做

給每個標籤一組候選位（曲線尾端的上／下／左／右、或沿曲線取樣數個 $x$），選與「已擺元素」重疊最小者，減少手填 label_point。這正是 CODE2VIDEO_STUDY.md 已提到的 occupancy-table 概念。

為何先不做：較複雜、較投機（Karpathy §2「簡單優先」）。B.1 先把「壞了會被抓到」做起來，自動擺位是「讓它更少壞」的加值，等 B.1 上線、看實際還剩多少手調量再決定。

B.3　最小版：只成文「多曲線圖建議顯式 label_point」

純文件，不改 code。把現行最佳實務寫進 DESIGN.md。不建議單獨採用——它沒補上 §1.4 的偵測盲區，問題照樣只能事後抓。可作為 B.1 的附帶文件。
