5 條 finding（Codex 建議 vs 實際採用）

Tighten[C1] 局部冗餘 · one_to_one_definition
name-then-define 重複：「…has a name: one-to-one. A function $f$ is one-to-one when…」——同一詞在緊鄰子句連說兩次（rubric 明列的 C1 tic）。
Codex 建議
“A function can be run backwards only when each output comes from exactly one input. We call $f$ one-to-one when different inputs always give different outputs — formally, $f(x_1)\ne f(x_2)$ whenever $x_1\ne x_2$.”
實際採用（已 lock）
“…each output comes from exactly one input, and that property has a name: one-to-one. Formally, that means different inputs always give different outputs — $f(x_1)\ne f(x_2)$ whenever $x_1\ne x_2$.”
採變體：去掉重複的第二個「one-to-one」（消 C1 tic），但保留「has a name: one-to-one」這個課堂命名手勢（Codex 版把它收成「We call f one-to-one」，會弱化「這是要記的術語」訊號）。兩者語義相同。
Tighten[C3] 讀順度 · finding_the_inverse_strategy
分詞尾拖沓、念出來卡：「…the result reads $y=f^{-1}(x)$, that last swap being just the habit of…」。
Codex 建議
“…the result reads $y=f^{-1}(x)$ because we conventionally use $x$ for the inverse's own input.”
實際採用（已 lock）
“…the result reads $y=f^{-1}(x)$. That last swap is just the habit of using $x$ for the inverse's own input.”
採變體：同樣切斷分詞拖沓（拆成兩句），但保留口語的「just the habit」而非較正式的「conventionally」（方法論 §4 偏口語 register）。
Tighten[C1] 局部冗餘 · inverse_of_a_rational
三重重述同一件事：「comes back to $x$ … comes back to $x$ too … both round trips return the input」。
Codex 建議
“This time let us verify both directions: $f$ of the inverse and the inverse of $f$ each come back to $x$, exactly the composition identities at work.”
實際採用（已 lock） · 按 Codex 原文
“This time let us verify both directions: $f$ of the inverse and the inverse of $f$ each come back to $x$, exactly the composition identities at work.”
保留「both directions」（這正是本例「雙向驗證」的教學點），只去掉三重重述。
Optional[C4] 句長／可聽度 · inverse_function_definition
一句話一口氣扛了：correspondence ＋ 兩個引號問句 ＋ domain/range 互換。
Codex 建議
“The idea is simply to read the same correspondence backwards. Where $f$ asks “given the input, what is the output?”, the inverse asks “given that output, which input produced it?” The outputs of $f$ become the inputs of the inverse, and the domain and range trade places.”
實際採用（已 lock）
“The idea is simply to read the same correspondence backwards. Where $f$ asks “given the input, what is the output?”, the inverse asks “given that output, which input produced it?” So the outputs of $f$ become the inputs of the inverse, and the domain and range trade places.”
