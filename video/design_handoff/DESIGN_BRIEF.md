# 微積分教學影片 — 視覺設計 Brief（交接給設計工具）

> 這份文件給設計師 / 設計工具（如 claude.ai Artifacts）閱讀，目的是設計一套教學影片的視覺語言。
> 文件最後有「要設計的清單」。`screenshots/` 資料夾是目前成品的實際畫面。

---

## 1. 這是什麼專案

把一本大學微積分教科書（LaTeX 講義）做成一系列**教學影片**，一節課一支，最終會有旁白語音（之後才接）。
影片本質是「**一堂課**」，不是課本朗讀——細節優先、口語、視覺帶著教。

- **觀眾**：大學微積分學生（台灣 NTU MATH 等級）。
- **語言**：旁白英文，數學符號標準 LaTeX。
- **目前範例章節**：Section 1.1 反函數（Inverse Functions）。
- **產出**：每節一支 1080p、16:9 的 MP4。

---

## 2. 重要技術約束（設計前必讀）

影片是用 **Manim**（Python 數學動畫引擎）算圖渲染的，不是網頁。這決定了什麼能設計、什麼不能：

| 你設計的東西 | 我們能不能用 | 說明 |
|---|---|---|
| **配色、字體比例、間距、版面、視覺層次** | ✅ 完全可用 | 這是最有價值的產出，會直接編進渲染主題。 |
| **靜態場景 mockup（一張圖說明某場景長怎樣）** | ✅ 當藍圖照做 | 請盡量提供。 |
| **SVG 圖示 / 裝飾元素** | ⚠️ 部分可用 | Manim 能載入 SVG，但漸層、陰影、網頁字體不保證。 |
| **CSS / JS / React 動畫、互動** | ❌ 無法轉換 | 動畫一律在 Manim 重寫，這是工程端的工作，不是設計端。 |

**請這樣分工**：
- **設計端負責「長什麼樣」**（靜態風格：色、字、版面、片頭樣貌、卡片/強調樣式）。
- **工程端負責「怎麼動」**（所有動畫：淡入、寫字、曲線描繪、強調閃光等，我們在 Manim 實作）。

**三條鐵則**：
1. **數學一律走 LaTeX**，不要在設計工具裡排版數學式（`f(x)`、分數、根號…）。你可以畫「這裡放一條數學式」的佔位框，實際數學由 Manim 的 LaTeX 算。
2. **畫布固定 16:9，1920×1080**。所有版面請以這個比例設計。
3. **深色背景**是目前基調（見下），如果你要改成淺色或其他，請整套一起重新定義（背景、各種強調色的對比都要重算）。

提供格式偏好：**截圖 / mockup 圖片**最直接（我們看得到圖）；**附上色票 hex 與字體比例的說明文字**更好；**SVG 原始碼**最理想（精確規格甚至可直接擷取）。

## 3. 目前的設計系統：「Midnight Canvas」

這是目前用的視覺語言。**你可以沿用並美化它，也可以提案全新風格**——但若全換，請整套重新定義。

### 設計理念
- **深色畫布、發光的數學、零冗餘裝飾**（dark canvas, luminous mathematics, zero chrome）。
- 「內容本身就是設計」：不用卡片、邊框、徽章堆砌。唯一的裝飾是標題下的一條細線。
- 每一刻畫面只有一個焦點；舊內容淡退、新內容寫上。

### 配色（hex）
| 角色 | 色碼 | 用途 |
|---|---|---|
| 背景 background | `#0b0c10` | 深藍黑畫布 |
| 主色 primary | `#e8e8f0` | 標題、中性文字（米白） |
| 次色 secondary | `#4cc9f0` | 定義、命題（冷青） |
| 強調 accent | `#f9a825` | 定理、重點（暖金） |
| 數學 math | `#7df9ff` | 數學式（電光藍） |
| 警告 warning | `#ff6b6b` | 反例、陷阱（珊瑚紅） |
| 成功 success | `#06d6a0` | 驗證、QED（翡翠綠） |
| 內文 text | `#c8c8d8` | 一般敘述（淺灰） |
| 淡化 muted | `#9aa0b5` | 已退場的舊內容（石板灰） |

### 字體與比例
- 字體家族：襯線體（serif，目前是 LaTeX 預設 Computer Modern 風格，給人「教科書 / 學術」感）。**如果你想換字體家族，這是可以討論的重點之一。**
- 字級（相對比例，實際渲染會等比縮放）：大標題 44 / 內文 30 / 數學 38 / 小標 24 / 圖標籤 26。
- 版面：左右留白寬（內容寬度約佔畫面 77%），呼吸感重要。

### 動畫語言（目前；供你理解節奏，動畫由工程端做）
- 數學式：像粉筆一樣「寫」出來（逐字描繪）。
- 曲線：由左到右「描繪」出來。
- 重點：閃一下金光強調。
- 舊步驟：淡退成石板灰，不刪除。
- 片頭/片尾：花俏一點（標題寫出+爆光、線條展開掃光、重點逐條滑入）。內容場景：克制、專注。

## 5. 要設計的清單

影片由「**場景模板（scene templates）**」組成，每種模板是一種版型，會在很多節課重複使用。請針對下列每一項，提供**版面 mockup + 任何配色/字體調整建議**。

### A. 全域視覺語言（最優先，先定這個）
- [ ] **整體風格定調**：沿用並精修 Midnight Canvas，或提出全新風格（若全新，需重定義背景＋全部強調色）。
- [ ] **配色系統**：9 個語意角色的色票（背景/主/次/強調/數學/警告/成功/內文/淡化），確認在深色背景上對比足夠。
- [ ] **字體系統**：標題、內文、數學的字體家族與大小比例（數學維持 LaTeX，但其字級/顏色要和整體協調）。
- [ ] **版面網格**：16:9 下的安全邊距、標題區、內容區、底部區的劃分。
- [ ] **強調樣式**：如何標示「重點數學式」（目前是閃金光）、如何表示「舊內容退場」。

### B. 片頭 / 片尾（視覺最重、最適合你發揮，建議優先設計）
- [ ] **片頭 intro**：章節開場卡版型（章節號 + 大標題 + 一句引導問題）。可加入圖形/裝飾元素（SVG 可用）。
- [ ] **片尾 outro**：總結卡版型（標題 + 數條重點 + 下一節預告）。

### C. 內容場景模板（數學重，風格參考你的設計，數學內容走 LaTeX）
- [ ] **definition_math**：定義 / 定理 / 命題（一句敘述 + 數學式）。**已有雛形**（見截圖），可直接美化。
- [ ] **example_walkthrough**：例題逐步解（左側步驟列 + 右側演算式 + 一句結論）。*尚未設計*。
- [ ] **procedure_steps**：流程 / 演算法（編號大數字 + 步驟 + 下方公式）。*尚未設計*。
- [ ] **theorem_proof**：定理 + 證明（敘述 + 證明步驟 + QED 結尾符號）。*尚未設計*。
- [ ] **section_transition**：小節之間的過場卡（大字幕 + 下一段預告）。*尚未設計*。
- [ ] **recap_cards**：節末重點回顧（數條重點 + 1~2 條要記的公式）。*尚未設計*（和 outro 類似但更內容導向）。

### D. 圖表場景（最特殊，需單獨討論）
- [ ] **graph_focus**：全畫面數學函數圖（座標軸 + 曲線 + 標記點 + 虛線 + 註解）。
  - 用途：每當課文用文字講一個「幾何/圖形主張」（鏡射、相交、兩個交點、形狀），就畫一張圖。
  - 設計重點：座標軸樣式、曲線配色（主曲線/對比曲線/輔助線各一色）、標記點（實心 vs 空心代表不同數學意義）、註解文字框樣式。
  - **這部分曲線本身由 Manim 算，但「視覺風格」（軸的粗細顏色、格線、標籤擺位、配色搭配）需要你定。**

### E.（可選）品牌元素
- [ ] 片頭前的 logo / 開場動畫（如果想要品牌識別）。
- [ ] 統一的浮水印 / 角落標記（如課程名）。
- [ ] 背景音樂風格方向（片頭/片尾用，之後接）。

---

## 6. 建議的討論順序

1. 先定 **A 全域視覺語言**（這是地基，定了其他才有共同標準）。
2. 再做 **B 片頭/片尾**（視覺發揮空間最大，回饋最快）。
3. 然後 **C 內容模板**（一個一個來，definition_math 已有雛形可改）。
4. 最後 **D graph_focus**（最特殊，單獨深談）。

## 7. 給設計工具的一句話總結

> 設計一套 **16:9、深色（可提案改）、襯線學術風**的數學教學影片視覺系統。重點交付：**配色票 + 字體比例 + 各場景版面 mockup**。數學式一律是 LaTeX 佔位（別自己排數學），所有動畫由工程端用 Manim 實作（別做 CSS/JS 動畫）。先定全域風格，再做片頭片尾，再做內容模板，最後談函數圖表。現況截圖見 `screenshots/`。
