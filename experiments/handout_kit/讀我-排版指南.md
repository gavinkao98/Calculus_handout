# 微積分講義 — 排版套件（Handout Kit）

這是我們這次排版設計的「可重複使用版」。共用的樣式與引擎只做一次，
之後每一章只要**複製範本、改一個設定區塊、寫小節內容**就能排出同樣質感的講義，
同時產出**線上閱讀版**（可即時調樣式）與**A4 列印版**（自動分頁、可存 PDF）。

---

## 1. 資料夾結構

```
handout-kit/
├─ 讀我-排版指南.md            ← 你正在看的這份
├─ template-screen.html       ← 線上閱讀版範本（複製這個排新章）
├─ template-print.html        ← A4 列印版範本（複製這個排新章）
├─ new-chapter-preview.html   ← 空白骨架的預覽（先打開看看效果）
│
├─ shared/                    ← 共用引擎與樣式（整包共用，通常不用動）
│  ├─ skin-hs.css             ·  語意配色 + 各種環境（定義/定理/範例…）的外觀
│  ├─ theme.css               ·  線上版 Tweaks 圖層（深色、密度、字級…）
│  ├─ figs.css                ·  插圖排版 + booktabs 數值表
│  ├─ print.css               ·  A4 紙張規格、頁首頁尾、列印顏色
│  ├─ plot.js                 ·  函數繪圖引擎（產生 SVG）
│  └─ tweaks-panel.jsx        ·  線上版的 Tweaks 面板
│
├─ new-chapter/              ← 「空白章節骨架」：複製它來排新的一章
│  ├─ figures.js              ·  這一章的插圖登錄表（內含一個範例圖）
│  ├─ sec-intro.html          ·  章節開頭範本
│  ├─ sec-markup-reference.html ·  ★標記速查★：示範所有可用環境，可整段複製
│  └─ sec-summary.html        ·  章末總結範本
│
└─ example-ch01/             ← 完整成品參考（就是第 1 章，可對照學寫法）
   ├─ figures.js
   └─ sec-intro.html … sec-summary.html
```

---

## 2. 三個核心觀念

1. **一章 = 一個資料夾。** 裡面放這一章的 `figures.js` 和一堆 `sec-*.html` 小節檔。
2. **內容用「語意標記」寫，不直接寫樣式。** 你只標「這是定義、這是定理、這是提醒」，
   顏色、間距、字型都由 `shared/` 自動套用。要全書改風格，只改 `shared/`，每一章一起變。
3. **同一份內容，兩種輸出。** `template-screen.html`（螢幕閱讀，含 Tweaks）和
   `template-print.html`（A4 列印）讀的是**同一批**小節檔，不必維護兩份內容。

---

## 3. 排一個新章節（步驟）

### 步驟 1 — 複製骨架資料夾
把 `new-chapter/` 複製一份，改名成例如 `ch02/`。

### 步驟 2 — 寫小節內容
編輯 `ch02/` 裡的 `sec-*.html`。每個檔案就是一個小節，外層一律是：

```html
<article class="sec" lang="en">
  <header class="sec-head">
    <h2 class="sec-title"><span class="sec-no">2.1</span>小節標題</h2>
  </header>
  <p>內文……行內數學寫 \(a^2+b^2=c^2\)，獨立公式用 \[ … \]。</p>
</article>
```

> **打開 `new-chapter/sec-markup-reference.html` 對照**——裡面把「所有」可用的環境
> （定義、定理＋證明、命題、系理、範例＋解答、提醒、策略、練習、插圖、表格…）
> 都寫了一遍，需要哪個就整段複製過去改字。

可自由增刪小節檔，檔名自取（建議 `sec-2-1.html`、`sec-2-2.html`…）。

### 步驟 3 — 登錄這一章的插圖（如果有圖）
在小節 HTML 放一個佔位：

```html
<figure class="figure" data-fig="my-plot">
  <figcaption><span class="fig-no">Figure 2.1</span> 圖說。</figcaption>
</figure>
```

再到 `ch02/figures.js` 用同一個 id（`my-plot`）登錄怎麼畫。
裡面已有一個 `demo-parabola` 範例，照著改即可。**沒有圖的章節，這個檔可以留著不管。**

### 步驟 4 — 接上範本
- 複製 `template-screen.html` → 改名（例如 `Chapter 2.html`）。
- 打開它，找到最上面的 **`CHAPTER 設定區塊`**，改三件事：

```js
const CHAPTER = {
  dir: "ch02",                                  // ← 你的章節資料夾
  brand: "Chapter 2",                           // ← 導覽列顯示的字
  fragments: ["sec-intro", "sec-2-1", "sec-2-2", "sec-summary"], // ← 小節順序
  macros: {},                                   // ← 自訂 KaTeX 巨集（可留空）
};
```

- 列印版同理：複製 `template-print.html`，改它的 `CHAPTER` 區塊
  （多一個 `runningHead` 欄位，是第 2 頁起的頁首跑馬字）。

**就這樣。** 其他程式都不用碰。

---

## 4. 編號規則（重要）

所有編號都是**手動寫死的文字**——`<span class="sec-no">2.1</span>`、
`<span class="env-num">2.3</span>`、`<span class="fig-no">Figure 2.1</span>`。
系統不會自動編號，好處是你能完全控制、跨小節也對得起來；代價是搬動內容時要自己改號碼。

---

## 5. 語意配色對照（顏色 = 教學角色，別自己塗色）

| 環境 class | 角色 | 顏色 | 外觀 |
|---|---|---|---|
| `env-definition` | 概念 | 青 teal | 左側色條 |
| `env-theorem` / `env-proposition` / `env-corollary` | 結論 | 藍 blue | 左側色條，敘述自動斜體 |
| `env-example` / `env-solution` / `env-exercise` | 練習 | 綠 green | 左側色條 / 外框 chip |
| `env-remark` | 旁白 | 灰 gray | 細框、字級略小 |
| `env-proof` | 證明 | 從屬 | 無色條、低調 |
| `env-caution` | ⚠ 提醒 | 紅 red | **實心淺底框**（會跳出來） |
| `env-strategy` | 解題策略 | 紫 violet | **實心淺底框**（會跳出來） |

只有 `caution` 與 `strategy` 是實心方塊，所以真正的警告才會在頁面上「跳出來」。

---

## 6. 線上版的 Tweaks（給審稿/試樣式用）

打開 `template-screen.html`，右下角的 **Tweaks** 面板可即時調：
深色/淺色、主色、字級、行距、內文字體（Latin Modern / Modern）、對齊（左/兩端）、
密度（舒適/緊湊）、顯示或隱藏插圖。這些只影響螢幕呈現，不會改到內容。

---

## 7. 列印 / 存 PDF

打開 `template-print.html`，等右上角顯示「N pages · A4」後，按
**Print / Save as PDF**（或瀏覽器列印），在印表機選「另存為 PDF」。
分頁、頁碼、頁首都已內建；左邊界較寬，方便打孔/裝訂。
**建議用 Chrome / Edge 列印，邊界設「無 / None」。**

---

## 8. 數學與字型

- 數學用 KaTeX：行內 `\( … \)`、獨立列 `\[ … \]`。
- 內文與數學都用 Latin Modern（Computer Modern 的網頁版），和原始 LaTeX 講義一致；
  插圖裡的標籤也走同一套 KaTeX，所以圖上的符號和內文公式長得一模一樣。
- 字型、KaTeX 由 CDN 載入，**第一次開啟需要連網**。要完全離線發佈時再另外打包。

---

## 9. 常見問題

- **圖沒出現？** 檢查 `data-fig` 的 id 和 `figures.js` 裡登錄的 key 是否一字不差。
- **公式變成原始碼？** 多半是 `\(` `\)` 沒成對，或網路還沒載到 KaTeX，稍等或重整。
- **想全書換個主色或字體？** 改 `shared/skin-hs.css` 最上面的變數即可，全部章節一起變。
- **某個環境要微調樣式？** 也在 `shared/skin-hs.css` 改，不要在小節 HTML 裡寫 style。
